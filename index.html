🚢 Incoterms 2020

國際貿易術語解釋通則——11個術語，掌握風險與費用的分界點

什麼是 Incoterms？

Incoterms（國際商業術語）由 ICC（國際商會）制定，規範買賣雙方在貿易合同中的風險移轉點和費用分擔。Incoterms 2020 共 11 個術語，分為兩大類：

  • 適用任何運輸方式（7個）：EXW、FCA、CPT、CIP、DAP、DPU、DDP
  • 僅適用海運／內水運輸（4個）：FAS、FOB、CFR、CIF
💡 關鍵原則：術語越靠前（如 EXW），賣方義務越少；越靠後（如 DDP），賣方義務越多。核心是搞清楚三件事：誰負責主運輸、誰買保險、誰辦進口清關。

情境應用

三種最常見場景：FOB / CIF / DAP 怎麼選

FOB 裝運港：廣澤負責到裝船，客戶自行安排運輸保險。適合有自己貨代的歐美大客戶，或客戶指定船公司時。

CIF 目的港：廣澤負責到目的港並附保險，適合中小型客戶或需要廣澤一起安排物流時。

DAP 門到門：廣澤負責送到客戶倉庫門口，不含進口稅。適合目的地物流可控且雙方已確認服務能力時。
  • 集裝箱和空運不要習慣寫 FOB：貨交 CY 或承運人時就已移轉，應優先考慮 FCA 或 CPT。
  • 客戶要 EXW 要先確認：外國買方常無中國出口報關能力，實務上多要改成 FCA。
  • 答應 DDP 前先算清楚：DDP 含目的國進口稅與清關，成本遠超單純加運費。
常見錯誤

四組最常混淆

組合真正差異新人怎麼記
FCA vs FOBFCA 在交承運人時移轉；FOB 在裝上船時移轉。只要不是典型船邊／船上交貨，就先想 FCA。
CFR vs CIFCIF 比 CFR 多了保險責任。有「I」才是賣方買保險。
CPT vs CIPCIP 比 CPT 多了保險責任，且適用任何運輸方式。CP* 系列都是交承運人就移轉風險。
DAP vs DDPDDP 比 DAP 多了進口清關與稅金責任。只要碰進口稅，就先重新算成本再答應。
情境應用

報價前一定要問的五個問題

  1. 貨是海運、空運、快遞還是多式聯運？
  2. 客戶要自己控物流，還是希望賣方一起安排？
  3. 保險由誰買？若客戶要廣澤買，保額和條款有沒有指定？
  4. 目的地清關與進口稅由誰負責？客戶是否清楚 DAP 和 DDP 的差異？
  5. 合同、PI、L/C、貨代訂艙資料上的條款是否完全一致？
💡 Incoterm 不是「填一個縮寫」，它決定報價、保險、文件、放貨與索賠的整條責任鏈。條款寫錯，後面整串都會跟著錯。

Incoterms 2020 重要更新

  • FCA 新增：可約定裝船後出具 On Board B/L，解決 L/C 項下的文件問題。
  • DPU 取代 DAT：更清楚表明「目的地卸貨後」交貨。
  • CIP 保險升級：最低要求提升至 ICC(A) 全險；CIF 仍維持較基礎的 ICC(C)。
  • 自有運輸：FCA、DAP、DPU、DDP 明確允許賣方使用自有運輸工具。

💳 付款方式

從預付到賒帳，掌握每種付款工具的風險、流程、控制點與放貨節奏

先懂概念

先把付款放回整條交易流程

交易階段付款上要先確認什麼如果沒先確認，後面會卡在哪裡
報價前客戶接受哪種付款工具、是否能收訂金、是否有信用條件報價只報單價，最後才發現條件談不攏
PI / 合同訂金比例、尾款時點、銀行費用、收款主體、幣別工廠可排產，但財務與客戶對付款理解不同
排產前訂金是否實際到帳、L/C是否審完、是否有內部授權先做貨再追款，主動權被拿走
出貨前尾款是否到帳、能否電放、是否需正本B/L或銀行交單貨到港卻不敢放貨，或先放貨後收不到錢
帳期中對帳窗口、到期日、催款節奏、逾期停單規則逾期後才發現對帳資料不齊，催收變被動
💡 新人最容易把付款看成「財務的事」。其實付款條件一開始就決定了你能不能排產、能不能放貨、能不能把尾款收回來。
先懂概念

樣品費、模具費、量產貨款要分開設計

收款項目常見做法控制重點
樣品費T/T 100% 預付，或至少先收打樣成本。先收再做，避免免費反覆改樣。
模具費50/50、40/60，或首批量產前收清。把模具 ownership、驗收節點與修改責任寫進 PI / 合同。
量產貨款T/T 30%+70%、即期 L/C、D/P、OA 等。尾款條件一定要和放貨條件綁定。
開發費 / 測試費視專案另列，不要默認含在單價裡。PPAP、實驗與第三方測試責任要先講清楚。
新手最常犯錯
  • 為了拿詢盤，把樣品、快遞與修改次數全部放成沒有界線的免費項目。
  • 只收部分模具費就默認開始大改模，後面容易被客戶綁住修改責任。
  • 合同只寫 `T/T` 卻沒寫清楚是裝船前、到港前還是到港後付款。
  • 前期費用沒收，最後想靠量產攤提，卻可能根本接不到量。
先懂概念

主要付款方式風險對比

方式風險平衡適用場景
T/T 預付 100%賣方風險最低，買方風險最高。小額、新客戶樣品或首次測試單。
T/T 30%+70%賣方風險低，前提是尾款到帳後才放貨。外貿最常見的平衡型結構。
即期 L/C雙方風險都可控，但單證要求高。新客戶大額訂單或需銀行信用支持時。
D/P 即期風險中等，單據還有控制力但拒付成本不低。熟悉客戶、能承受目的港風險時。
D/A 遠期買方先拿貨，賣方追款風險明顯升高。有信用額度、歷史交易與風控配套時。
OA 賒帳本質是信用銷售，壞帳風險最高。長期穩定老客戶，且有額度與催收機制時。
先懂名詞

付款縮寫全名速查

T/T Telegraphic Transfer

電匯。外貿最常見的銀行匯款方式，可做100%預付，也可做30%訂金+70%尾款。

L/C Letter of Credit

信用狀/信用證。銀行按單據付款，不是看貨物本身；核心是單證相符。

D/P Documents against Payment

付款交單。買方付款後，銀行才交出提單等單據給買方提貨。

D/A Documents against Acceptance

承兌交單。買方承諾未來付款後即可取得單據，賣方風險高於D/P。

OA Open Account

賒帳。先出貨、後付款，通常只適合長期穩定且有信用額度管理的客戶。

SINOSURE China Export & Credit Insurance

中國信保。不是付款方式，而是接受OA或遠期帳期時可搭配的信用風險工具。

常見錯誤

T/T 水單、實際到帳、電放、正本 B/L 是四件不同的事

名詞代表什麼可不可以當成交付依據
Bank Slip / 水單客戶說已付款的憑證或截圖不能。只能代表客戶說已匯，不代表公司已收到。
到帳公司帳戶實際收款，財務可確認金額與幣別可以。排產、出貨、放貨應以到帳為準。
Telex Release / 電放通知目的港可不憑正本 B/L 放貨風險高。電放前要先確認付款條件已滿足。
Original B/L / 正本提單控制提貨的重要單據寄出或交出前，要先確認尾款、D/P或L/C條件都已成立。
⚠️ 很多放貨事故都不是因為文件不懂，而是把「客戶說已付」誤當成「公司已收到」。真正的分界點永遠是財務確認到帳。
情境應用

六種付款方式實際怎麼跑

T/T預付100%
Telegraphic Transfer
流程

客戶先付款 → 財務確認到帳 → 業務通知排產或出貨。

控制點

確認實際到帳，不只看水單或客戶截圖。

風險

買方風險高，通常只適合樣品、小額、新客戶首單。

T/T 30%+70%
Telegraphic Transfer
流程

30%訂金到帳 → 排產 → 出貨前收70%尾款 → 尾款到帳後放貨。

控制點

尾款未到帳前，不電放、不寄正本B/L、不放貨。

風險

最大風險是業務因客戶催貨而先放貨，導致尾款失控。

即期L/C
Letter of Credit
流程

客戶開證 → 廣澤審證 → 出貨 → 交單 → 銀行審單 → 單證相符付款。

控制點

先審證再出貨，文件按L/C逐字核對。

風險

不符點會造成扣費、延遲付款，嚴重時可能被拒付。

D/P即期
Documents against Payment
流程

廣澤出貨 → 單據交銀行託收 → 買方付款 → 銀行交單 → 買方提貨。

控制點

正本單據在銀行手上，買方未付款前不應取得提貨單據。

風險

買方拒付時，貨已到港，賣方面臨滯港費、退運或轉賣壓力。

D/A遠期
Documents against Acceptance
流程

廣澤出貨 → 買方承兌遠期付款 → 銀行交單 → 買方提貨 → 到期付款。

控制點

只適合有信用額度、歷史交易和管理層批准的客戶。

風險

買方先拿貨、後付款，到期不付時賣方追款被動。

OA賒帳
Open Account
流程

先出貨 → 客戶收貨入庫 → 月結或帳期付款 → 財務對帳與催收。

控制點

設定信用額度、帳期、逾期停單規則，必要時搭配信保。

風險

壞帳、拖款、折扣爭議與對帳扯皮都會直接影響現金流。

💡 看付款方式時，不要只看誰先付錢，更要看三件事：誰控制提單、誰控制放貨、如果客戶不付錢你還剩下什麼籌碼。
先懂概念

L/C（信用狀）操作要點

L/C = 銀行信用代替買方信用，單證相符即付款。

  • 收到L/C：48小時內仔細審核每一個條款
  • 常見不符點：描述不符、金額超額、日期問題、缺少要求的聲明
  • 發現問題：立刻申請Amendment（修改），不能帶著已知不符點出貨
  • UCP600：L/C的國際規則，銀行有5個工作日審核單據
⚠️ 軟條款警告：若L/C包含「需買方出具合格證才付款」等條款，等於架空了L/C保障，需要立刻要求刪除！
情境應用

L/C審證Checklist

  • 開證人、受益人、通知行、金額、幣別、有效期是否正確
  • 最遲裝運日、交單期限、信用狀到期地是否可執行
  • 品名、規格、數量、單價、貿易條件是否和PI完全一致
  • 要求提交的文件是否合理，例如CI、PL、B/L、保險單、產地證、檢驗證明
  • 提單要求是否可操作，例如是否要求Full Set、Clean On Board、To Order、Notify Party寫法
  • 銀行費用條款是否清楚，避免出貨後才發現不合理費用由賣方承擔
常見錯誤

L/C最常見的不符點

類型常見情況如何避免
描述不符CI、PL、B/L上的品名與L/C不同直接複製L/C文字，不自行簡化或改寫
日期問題晚裝船、晚交單、文件日期前後矛盾提前倒排交單時間，不要壓線出貨
單據抬頭錯誤Consignee、Notify、開證申請人拼寫不同用同一份核對表檢查所有單據
數量/金額偏差發票金額、件數、重量與L/C限制不符出貨前和工廠、物流、財務最後對表
聲明缺漏少了L/C要求的特殊聲明或保證語句做交單模板，不要臨時補寫
常見錯誤

不能接受的L/C軟條款

  • 需要買方簽字、買方驗貨通過或買方出具文件後銀行才付款
  • 文件內容依買方最終確認為準，等於把付款控制權交回買方
  • 要求提供賣方無法自行取得的第三方文件，且無明確出具機構
  • 過短的裝運或交單期限，實務上根本無法完成
  • 寫法模糊，例如「clean documents acceptable as determined by applicant」
⚠️ 判斷標準很簡單：如果付款條件最後還是取決於買方一句話，那它就不是你以為的「安全L/C」。
情境應用

匯率風險管理

  • 以USD報價時，人民幣升值=利潤減少
  • 應對：報價加1-2%匯率緩衝、縮短報價有效期、遠期結匯鎖匯率
  • 出口退稅率影響實際收益，報價前需確認
情境應用

付款條件選型建議

客戶情況建議條件為什麼
新客戶 / 打樣單T/T 100% 預付先把首次交易風險壓到最低。
新客戶 / 大額量產T/T 30%+70% 或 即期 L/C兼顧客戶接受度與收款安全。
熟悉客戶 / 長期合作D/P 或有限額 OA成交率會更高，但前提是內部已有信用管理。
高風險市場 / 首次遠期付款優先即期 L/C避免整張單只靠買方信用撐住。
補強措施
  • 新客戶先收款再安排樣品或排產，不要被「先做給我們看」帶著走。
  • `T/T 30%+70%` 的關鍵不是條件漂亮，而是訂金到帳後才排產、尾款到帳後才放貨。
  • 只要進入 `D/P`、`D/A` 或 `OA`，就要同步設定信用額度、對帳節奏與信保配套。
  • 即期 `L/C` 也不是萬靈丹，仍要先審核開證銀行和條款。
先做控制

收款節點與出貨紅線

  1. 確認PI、付款條件、受益人帳戶資訊完全一致
  2. 收到訂金且確認實際到帳，再通知工廠排產
  3. 出貨前先準備箱單、發票草稿、照片或驗貨資料給客戶確認
  4. 尾款條件是裝船前付款時，必須先確認全額入帳，再安排電放或寄正本單據
  5. L/C條件下，先完成審單與改證，再安排出貨與交單
⚠️ 紅線：未到帳不電放、未到帳不寄正本B/L、條件未確認不承諾OA。這三條守住，很多壞帳和扯皮都能避免。
先做控制

放貨控制矩陣

情況能不能放貨備註
T/T尾款未到不能不能電放、不能寄正本、不能口頭答應先放
L/C條款未改好原則上不能安排出貨先改證，再出貨，否則後面不符點風險極高
D/P條件下買方尚未付款銀行不應交單但賣方仍要預想客戶拒付後怎麼處理貨物
OA在信用額度內可放，但需照授權流程要有對帳、逾期追收與信保配套
客戶要求先電放後付款高風險，不建議除非已有明確批准與足夠保障
常見錯誤

新客戶風控清單

  • 確認公司名稱、註冊地址、官網、聯絡人職稱與公司郵箱是否一致
  • 要求客戶提供完整開票資訊、收貨資訊、付款主體資訊
  • 大額訂單先做背景調查：是否有壞帳紀錄、是否真有採購權
  • 如果要求改收款帳戶、急單、異常高價或異常寬鬆OA，要提高警覺
  • 必要時搭配SINOSURE或內部信用額度管理，不要只靠感覺判斷
情境應用

應收帳款與催款節奏

  1. 帳期開始前：先確認Invoice、付款到期日、付款路徑與對帳窗口
  2. 到期前7天：做第一次友善提醒，確認是否已安排付款
  3. 到期當天：再次確認匯款狀態，保留書面紀錄
  4. 逾期7天內：升高追蹤頻率，要求明確付款日期與原因
  5. 逾期超過30天：視內部政策停止新單、停止出貨或啟動管理層介入
💡 催款不是等過期才開始。真正有效的催款，是在到期前就把對帳、文件、付款窗口全部理清，讓客戶沒有拖延藉口。
先做文件化

銀行費用與付款細節，也要寫在PI裡

  • 付款條件要寫完整，例如 `30% deposit, 70% before shipment`，不要只寫 `T/T`
  • L/C要寫清楚是否接受分批裝運、轉運、交單天數、銀行費用承擔方
  • 如果客戶分兩個主體付款，要先確認內部是否接受，以及文件如何對應
  • 付款幣別要跟報價幣別一致，不要等出貨後才臨時改成人民幣或其他幣別
  • 涉及手續費時，事先講清楚 `bank charges outside issuing bank are for applicant` 這類分配原則
情境案例

付款情境案例：遇到客戶這樣說怎麼辦

案例1：新客戶要求OA 60天

客戶說：「我們所有供應商都是OA 60 days，你們也要配合。」這時不能因為客戶看起來大就直接答應。

建議處理：先回到T/T 30%+70%或即期L/C；若客戶堅持，要求信用資料、交易紀錄、信保評估和內部授權。

案例2：尾款未到，客戶要求先電放

客戶說：「貨已到港，我們急著清關，你們先Telex Release，我們明天付款。」這是典型放貨風險。

建議處理：禮貌但明確拒絕，說明公司政策為尾款到帳後電放；可提供單據副本協助客戶預先清關準備。

案例3：L/C含買方驗貨通過才付款

這種條款表面上是L/C，實際上付款仍掌握在買方手上，銀行保障被削弱。

建議處理：要求amendment刪除軟條款，改成客觀文件要求，例如第三方檢驗證書或出貨前檢驗報告。

案例4：D/P貨到港後買方拒付

買方拒絕付款，銀行不交單，但貨已在目的港，滯港費和倉租每天增加。

建議處理：立即確認目的港費用、貨物可轉賣性、退運成本；同步要求客戶書面說明，必要時尋找備用買方。

案例5：D/A到期後客戶拖款

買方已承兌並提貨，到期後卻以「財務流程」「客訴未結」為由拖延付款。

建議處理：先對帳並固定付款承諾日期；逾期仍未付時，暫停新單或出貨，升級到管理層與信用保險流程。

案例6：客戶通知更改收款帳戶

任何收款帳戶變更都可能涉及詐騙、郵箱被盜或內部流程錯誤，不能只憑郵件執行。

建議處理：用已知電話或原有聯絡渠道二次確認；付款資料變更需財務與主管核准，不接受臨時口頭變更。

⚠️ 付款問題的核心不是「客戶怎麼說」，而是「公司還剩多少控制權」。只要涉及放貨、OA、遠期、改帳戶，都要先升級確認。

三種常見交易場景，文件包長不一樣

先懂概念

七大核心單據

  • PI（Proforma Invoice）：接單前的商務確認文件，後面很多正式文件都會照它走。
  • CI（Commercial Invoice 商業發票）：正式結算文件，描述貨物、金額、Incoterm與付款條件。
  • PL（Packing List 裝箱單）：每箱明細，影響報關、查驗、客戶收貨與倉庫點數。
  • B/L（Bill of Lading 提單）：海運核心文件，也是控制放貨的重要籌碼。
  • CO（Certificate of Origin 原產地證）：證明商品產地，關係到優惠關稅與客戶進口成本。
  • Insurance Policy / Certificate：CIF / CIP下常會要求，金額與條款不能亂。
  • L/C交單文件：不是單一文件，而是一套按信用狀條件準備的文件組合。

責任分工提醒
  • 接單前通常由業務主導，但一進到出貨前就會和工廠、單證、物流一起共用同一套資料。
  • `B/L Draft` 雖然多半由貨代出稿，最終被客戶看到與拿去清關的責任仍會回到業務身上。
  • 遇到 `L/C` 或銀行交單時，業務、財務與銀行之間的版本控管必須非常嚴格。
先懂概念

提單（B/L）是文件章最重要的一張

  • 正本提單：通常3份正本，任一份被用來提貨後，其餘即失效。
  • 電放（Telex Release）：速度快，但放棄了正本B/L控制；一定要確認付款或內部批准條件。
  • Clean B/L：L/C下常要求清潔提單，不能有不利備註。
  • To Order B/L：可背書轉讓，L/C與控制貨權時常見。
  • 最常出錯：Consignee、Notify、Freight Collect / Prepaid、港口名稱、件數重量。
💡 誰控制正本B/L，誰通常就還保有最後一道放貨槓桿。所以文件錯不只是麻煩，而是可能直接失去收款控制。
常見錯誤

原產地證不是每票都一樣

類型用途新人先確認什麼
一般 CO證明產地，但通常不含優惠稅率。客戶只是要產地證，還是要關稅優惠。
Form E對應中國 - 東盟 FTA 優惠。目的國是否適用，產品是否符合規則。
RCEP CO對應 RCEP 成員國優惠。材料來源與原產地規則能不能成立。
其他 FTA 文件依客戶目的國或貿易協定要求。不要先答應，先確認公司能不能做。
出具機構提醒
  • 一般 `CO` 常見由貿促會或商會出具；`Form E`、`RCEP CO` 則常見由海關或商務系統處理。
  • 真正的重點不是背機構，而是先判斷客戶要的是「普通產地證」還是「優惠關稅文件」。
常見錯誤

CI / PL / B/L 最容易不一致的地方

欄位常見錯誤後果
品名描述PI寫法、CI寫法、L/C寫法不一致銀行不符點 / 客戶清關卡關
件數與箱數工廠改箱數，但PL沒更新客戶收貨對不上 / 查驗麻煩
毛重/淨重報關數據和PL不一致海關、貨代、客戶都會追
收貨人抬頭Consignee / Notify 拼字或公司名錯提貨延誤甚至重簽文件
港口與IncotermPI、B/L、L/C條件彼此不一致文件和交易條件衝突
寄出前檢查

交單前Final Check Checklist

  • CI、PL、B/L上的品名、數量、件數、毛重、淨重是否彼此一致。
  • 客戶名稱、地址、Consignee、Notify Party拼寫是否完全正確。
  • Incoterm、裝運港、目的港是否與PI / L/C一致。
  • 日期順序是否合理：開票日、裝船日、保險日、產地證日期不要互相打架。
  • 若為L/C，描述、標點、大小寫、附件聲明都要按信用狀要求處理。
  • 正本寄出或電放前一定先確認B/L draft，避免貨到港才發現抬頭錯誤。
💡 最常見錯誤不是大錯，而是小錯累積：品名少一個字、地址少一行、箱數改了但PL沒更新。建立交單前最後一輪對表非常重要。

💬 詢報價技巧

從詢盤到PI、從PI到訂金、從訂金到排產，建立一條能落地的成交流程

先懂概念

詢盤到排產：完整閉環

階段你要做什麼關鍵交棒
收到詢盤先確認客戶身份、產品類型與資料完整度。先回信並列補件清單，不要急著報價。
內部評估找工程、採購、工廠確認可行性與成本。先決定可做、不做或有條件做。
正式報價一起報單價、Incoterm、付款條件、交期與有效期。讓客戶知道你報的是整體交易條件，不只是價格。
客戶確認跟進價格、規格、模具、付款與文件要求。把所有口頭條件改成修訂版 PI 或會議紀要。
接單確認 PI、訂金、模具費、樣品費與排產條件。未達收款條件前，不要進入排產與備料。
排產啟動把資訊交接給工廠、單證、物流與品質。交期、文件與付款條件要從一開始就對齊。
輸出物提醒
  • 收到詢盤後至少要留下回信與補件清單；內部評估後要有 `可做 / 不做 / 有條件做` 的結論。
  • 正式報價後通常會形成 `Quotation` 或 `PI draft`，客戶確認後則要落成修訂版 `PI` 或 `MOM`。
  • 真正啟動排產前，至少要拿到正式 `PI` 與訂金憑證或其他內部授權依據。
情境應用

收到詢盤的黃金24小時

  1. 24小時內先回覆收到，即使完整報價還沒出來，也要讓客戶知道你在處理。
  2. 先判斷是標準件、客製件、發光件、模具新案，還是單純比價。
  3. 若資料不完整，不要用猜的報價；先列補件清單，比報錯價再改好太多。
  4. 告知完整報價需要的時間，例如 `2個工作日內提供正式回覆`。
  5. 報價寄出後3到5天要主動跟進，不要把沉默誤判成沒機會。
⚡ 新人最容易做錯的不是回太慢，而是「回得快但報得太早」。先把詢盤分類，再決定要不要進入正式報價。
先懂概念

層級你要寫什麼漏掉的風險

交易層Incoterm、裝運港/目的港、付款條件、有效期價格看似一樣，實際責任完全不同

情境應用

報價成本拆解範例

同一產品從工廠價推到成交價，不能只看材料和人工：

  • 產品成本：材料 22 + 人工/製造 6 + 管理/損耗 2 + 利潤 3 = USD 33
  • FOB加項：內陸運輸 0.8 + 報關/港雜 1.2 = FOB USD 35
  • CIF加項：海運 2.5 + 保險 0.2 = CIF USD 37.7
  • 還沒結束：若付款是OA、若客戶要求第三方驗貨、若包裝需升級、若目的地有301關稅，報價還要再往上調整
💡 報價不是只看工廠成本。至少要把付款風險、匯率、目的港要求、包裝升級、驗貨成本一起算進去，才能避免接單後才發現沒利潤。
先做文件化

PI（形式發票）是成交控制文件，不只是附件

  • 公司名稱、地址、聯繫方式與客戶名稱要準確，後面會一路帶到CI、PL、B/L與銀行文件。
  • 產品描述要和客戶圖紙、報價版本、L/C要求對得起來。
  • 數量、單位、單價、總價、Incoterm、裝運港/目的港、付款條件、交期、有效期都要寫清楚。
  • 模具費、樣品費、分批交貨、包裝升級、驗貨要求若有，也要在PI或附件裡寫出來。
  • PI一旦被客戶確認，很多條件就進入執行層，後續不能再當口頭討論。
先懂連動

詢報價和付款、單據其實是連在一起的

你現在報的條件後面會影響什麼
Incoterm影響運費、保險、提單抬頭、保險單、目的地責任
付款方式影響是否能排產、是否能電放、是否要控制正本B/L
交期影響工廠排程、訂艙、是否需要客戶修改L/C
文件要求影響CI/PL/B/L/CO/保險單的一致性和交單難度
➡️ 所以新手最好的習慣不是「先把價報出去」，而是從一開始就把 `Incoterm + payment + documents + lead time` 當成一包來報。
常見錯誤

不能亂承諾的四件事

  • 交期：沒有跟工廠確認產能前，不要直接承諾 exact date。
  • 索賠：沒有先掌握責任歸屬，不要先答應折扣、補貨或賠償。
  • 付款條件：OA、D/A、電放、先放貨後付款都要經內部批准。
  • 模具 / 開發費：模具所有權、維修責任、備份模具與折抵條件要寫清楚。

🏆 品質管理

把 IATF、APQP、PPAP、8D 變成業務真的用得上的節奏、文件與回覆方式

先懂概念

品質章不是背縮寫，而是知道哪個階段會被客戶問什麼

名詞客戶在意什麼業務至少要知道什麼
IATF 16949你有沒有進入汽車供應鏈的基本管理能力。這是進場門票，不是成交保證。
APQP新案導入是否有節點、風險控制與跨部門規劃。你要知道專案現在卡在哪一階段，而不是只回「還在做」。
PPAP量產前文件、樣件與製程證據是否齊全。至少分得清客戶是要樣品、試產件還是正式 PPAP 提交。
8D發生客訴後，你們能不能有效止血、找根因、避免再發。你不一定會寫，但一定要知道何時交初步報告、何時交完整報告。
先懂概念

APQP 五階段：業務在每一段要會接什麼球

階段客戶通常會問什麼業務要準備什麼
1. 立項 / 報價你們能不能做、交期多久、風險在哪裡。圖紙版本、工藝可行性、樣品週期與條件報價。
2. 設計與開發樣件、模具、材料與驗證計畫怎麼排。打樣節點、模具費、材料確認與驗證時間表。
3. 製程開發量產流程、治具、控制計畫是否穩定。和工廠確認 trial build、量產條件與風險點。
4. 驗證與 PPAP尺寸、材質、外觀與文件是否符合要求。客戶要求的 PPAP level、提交日期與缺件補交方式。
5. 量產與改善首批量產穩不穩、客訴如何關閉。出貨節奏、問題升級路徑與 8D 時程。
先懂概念

PPAP：業務不用會做，但要分得清客戶到底在要什麼

文件 / 輸出用途晚交會發生什麼
尺寸報告證明關鍵尺寸符合圖紙要求。客戶無法核准樣件，後面試產和 SOP 都會被卡住。
材質 / 測試報告證明材料與性能符合客戶或 OEM 標準。客戶可能不接受替代料，也不會放行量產。
控制計畫 / PFMEA證明製程風險與控制點已被規劃。審廠或 PPAP 評審時容易被認為量產風險過高。
樣件核准 / PSW代表客戶正式接受 PPAP 提交結果。沒有核准就提前量產，後面風險全在自己身上。
💡 新人最常混淆的是「客戶要樣品」和「客戶要 PPAP」。前者是看件，後者是看件加文件、加製程證據。
先懂概念

8D 問題解決流程

步驟核心任務業務要知道的重點
D1-D2組團隊、定義問題。先把問題說清楚，不要用模糊詞如 `有點異常`。
D3先做臨時對策止血。客戶最先要的是圍堵，不是長篇理論。
D4-D5找根因並提出永久對策。不要在根因未確認前先承諾賠償或責任歸屬。
D6-D7驗證對策有效並防止再發。客戶會看你有沒有把改善真正落到流程。
D8關閉問題並沉澱經驗。問題結案不等於客戶信任恢復，還要看後續表現。
情境應用

客訴節奏：24 小時、48 小時、10 天你要交什麼

時點你要交出去的東西業務重點
24 小時內收到通知、初步回覆、資料需求與下一次更新時間。先回應，不要失聯，也不要先認責。
48 小時內臨時對策、批次影響評估、是否停貨 / 分選 / 補貨。把止血方案講清楚，讓客戶知道風險已被控住多少。
10 天內完整 8D、根因、永久對策與防再發措施。如果趕不上，要提前說原因與新的提交時間。
常見錯誤

品質問題最容易把客戶激怒的 5 種錯法

  • 先承認責任：根因沒查清就說「一定是我們的問題」，後面很難收回。
  • 只說在確認：沒有時間點、沒有臨時對策，客戶會覺得你沒在動。
  • 先談折扣：用價格去蓋問題，會讓客戶覺得你根本沒找根因。
  • 沒先隔離：同批庫存、在途貨不先控住，問題會越滾越大。
  • 把 8D 當作文：如果內容只是在重述客訴，沒有行動與驗證，客戶不會接受。
情境應用

業務收到客訴後，前 2 小時要做什麼

  1. 先回覆客戶已收到資訊，承諾下一次更新時間，不要先失聯。
  2. 立即收集訂單號、批號、數量、不良照片 / 影片、發生站點、是否已停線。
  3. 通知工廠與品保，確認同批庫存和在途貨是否需要暫停出貨。
  4. 如果問題可能擴大，先給客戶臨時對策，例如全檢、隔離、替換方案。
  5. 內部約定 24 小時、48 小時、10 天的輸出節點，不要只說「我們在確認」。
情境應用

客戶常要求的品質資料

場景客戶會要什麼你最該先確認什麼
新項目導入PPAP 文件包、尺寸報告、材質與檢驗標準。客戶要的 level、提交日期與缺件能否分階段補交。
量產穩定期控制計畫、巡檢紀錄、Cpk 或關鍵尺寸能力資料。客戶是例行追蹤，還是因為現場出現異常。
客訴發生時8D、分選報告、遏制措施與補貨計畫。先止血再根因，不要把節奏顛倒。
審廠 / 稽核認證證書、流程圖、培訓紀錄、追溯資料。客戶關心的是文件完整，還是現場執行一致性。
💡 業務不用自己會做 PPAP 或 8D，但必須知道客戶要的是什麼、何時要、晚交會影響什麼。這樣才不會把問題拖成信任危機。
情境案例

品質案例：客戶最常怎麼追，你要怎麼接

案例1：首批量產外觀刮傷

客戶已在產線發現刮傷，但還不知道是製程、包裝還是運輸造成。

先隔離同批、確認包裝與裝櫃記錄，再回覆 24 小時初步調查與 48 小時臨時對策。

案例2：客戶要 PPAP，但工廠只準備了樣品

這不是再寄一次樣品就能解決，因為客戶要的是量產證據。

先問清楚 PPAP level 與缺哪些文件，再拉工程、品保、工廠補齊節點。

案例3：客戶要求立刻索賠

客訴剛進來，根因還沒查清，但對方已要求折價或補貨。

先確認影響範圍與責任界面，回覆臨時處置節奏，不要在資料不足時先答應金額。

案例4：現場稽核前兩天才通知

客戶突然要看追溯、流程與改善紀錄，這時最怕文件版次混亂。

先把認證、流程圖、培訓紀錄、8D 案例與追溯演練資料收齊，再安排現場路線。

🚛 物流操作

把訂艙、報關、截關、提單、到港異常與客戶通知節奏真正串成一條線

先懂概念

物流章怎麼學：先看節點，再看異常

面向你要先搞懂什麼為什麼重要
時間點何時訂艙、何時截關、何時出 B/L draft。物流出問題，常常不是做錯，而是做得太晚。
資料流訂艙資料、報關資料、箱單發票與提單抬頭怎麼對齊。資料一旦不同步，後面文件與清關都會亂。
控制點尾款、放貨、B/L draft、客戶更新節奏。物流不是貨代的事，很多風險最後還是回到業務身上。
先懂概念

海運關鍵術語

縮寫全稱說明
FCLFull Container Load整箱，獨享一個集裝箱。
LCLLess than Container Load拼箱，多客戶共用。
S/OShipping Order訂艙確認或裝船許可。
ETD / ETAEstimated Time of Departure / Arrival預計離港 / 到港時間，不等於最終保證。
THCTerminal Handling Charge碼頭操作費，常依條款與港口慣例分攤。
D/ODelivery Order提貨單，通常憑正本 B/L 或電放指示換取。
先懂概念

出貨八個節點：你每一步要盯什麼

節點你要盯什麼最常出錯
接單後先對齊交期、Incoterm、付款與港口。業務先答應日期，工廠和物流根本來不及。
訂艙前確認箱數、材積、件數、截關日與船期。資料不完整就先訂艙，後面反覆改件數。
報關前HS Code、申報要素、毛淨重、箱數要一致。報關資料和箱單發票不同步。
送港前確認拖車、VGM、箱嘜、棧板與裝櫃照片要求。錯過 cut-off 或現場發現包裝不合規。
裝船後立刻追 B/L draft、船名航次、ETD / ETA。提單草稿沒校對就直接放出正本。
交單前CI、PL、B/L、付款條件要對齊。提單抬頭和 PI / L/C 不一致。
到港前把 ETA、放貨條件、清關準備通知客戶。客戶以為能提貨，實際文件或付款還沒好。
異常發生時先告知原因、影響、下一次更新時間。想等完全查清才說，結果客戶先被港口或工廠通知。
情境應用

貨代對接時，你至少要先交這些資料

資料類型至少要先給什麼不先給會怎樣
訂艙資料港口、件數、箱型、材積、毛重、預計出貨日。貨代無法鎖艙位，或先訂錯船型與時間。
提單資料Shipper、Consignee、Notify、Freight 條件。B/L draft 反覆修改，容易延誤放單。
報關資料HS Code、申報要素、箱數、毛淨重、發票金額。報關延誤，嚴重時會錯過截關。
客戶特殊要求箱嘜、棧板、拍照、驗貨、AMS / ENS 等。貨代和現場不知道要配合哪些附加要求。
情境應用

選港口策略

港口適合情況你要注意什麼
鹽田港直達航線多、歐美線需求高時。船期資源好，但拖車與排隊成本也要一起算。
南沙港華南整體平衡型出貨。看船期和截關安排，不要只看地理距離。
虎門或其他近港特定近洋線或本地便利性需求。航線可能較少，轉運與時效要先問清楚。
💡 選港口不能只看哪個近，還要一起看拖車費、直達船期、航行時間，以及客戶或 L/C 是否限制港口。
情境應用

FCL / LCL / 空運怎麼選

方式適合情況重點提醒
FCL貨量大、交期穩定。安全且時效相對穩，但未裝滿也要付整櫃費。
LCL貨量小、無法湊整櫃。前期成本低，但拼箱 / 拆箱會拉長時間並增加碰損風險。
空運急單、補貨或樣品。速度快，但成本高且尺寸重量限制多。
常見錯誤

物流異常時，第一步該做什麼

異常第一動作不要先做什麼
錯過截關先重新鎖新船期、重算 ETD / ETA，再通知客戶。不要等新艙位完全確定才第一次對外說。
海關查驗先確認查驗原因、估算延誤與費用影響。不要只說 `customs issue` 而沒有下一次更新時間。
B/L 草稿錯誤先卡住正本或電放，立刻要求貨代重出 draft。不要抱著僥倖想法說 `應該沒差`。
甩櫃 / 延船先拿到新船期與原因，再同步改文件節奏。不要只更新工廠，不更新客戶與單證。
包裝或裝櫃異常先拍照留證，確認是否要重包、補強或換箱。不要一邊出貨一邊再想怎麼跟客戶交代。
⚠️ 物流異常最怕「晚說」。就算暫時沒有完整解法，也要先告訴客戶：發生了什麼、影響哪一票貨、下一次更新時間是何時。
情境案例

物流案例：你最容易在哪裡被追著跑

案例1：客戶臨時改 Consignee

船已開，B/L draft 已出，但客戶突然要改收貨人資訊。

先確認是否尚未出正本，再看是否會影響清關、L/C 與費用，不能直接口頭答應。

案例2：裝櫃當天發現箱嘜版本錯

工廠印了舊版外箱標示，如果直接出貨，客戶倉庫可能拒收。

先拍照比對客戶最新版要求，判斷是重貼、重包還是可由客戶書面接受。

案例3：貨代說 ETA 延後 5 天

客戶工廠排程已經排進去，這時客戶真正要的是新的可執行節點。

回覆新的 ETD / ETA、原因、是否影響提貨與付款，並承諾下一次更新時間。

案例4：空運急單但尾款還沒到

業務最容易被交期壓力推著走，最後在物流節點把收款控制放掉。

先把付款條件講清楚，再決定能不能出空運，不要用急件當成先放貨理由。

🤝 談判技巧

不是只會回「我再申請看看」，而是知道先守什麼、能換什麼、何時該停

先懂概念

談判不是單純壓價，而是交換條件

概念代表什麼你怎麼用
BATNA談不成時你還有沒有其他可接受方案。先知道底線，才不會被對方一句話帶走。
ZOPA雙方可接受條件的重疊區間。如果根本沒有重疊，就不要假裝能成交。
錨定誰先定義價格與條件範圍，誰就先有主導權。報價時把 MOQ、付款、交期一起帶上，不只丟單價。
互惠每次讓步都應該換到對方某個明確條件。不要只會降價，要換量、換付款、換長約或換包裝條件。
先做準備

談判前準備表：沒準備好，現場就只會被追著跑

項目你要先準備什麼沒準備會怎樣
價格底線知道哪些成本能動、哪些不能動。客戶一壓價，你只能一直往內部追降。
可交換條件MOQ、付款、交期、包裝、模具、預測量。你只剩單價能談，談判很快失血。
不可承諾項OA、D/A、DDP、超短交期、先放貨後付款。現場講出去，回來很難收回。
替代方案標準版、降配版、分階段導入版或長約版。客戶卡在一個點時，你沒有第二條路可走。
先做原則

價格談判黃金原則

永遠不要接受第一個還價。
每一次讓步都要比前一次更小，讓對方感受到你接近底線。
讓步必須附帶條件，例如「如果年量提升到 80,000 pcs，我們可以重算單價」。
合同簽訂後再要求年降，不能只談價格，必須同步談量、付款或專案範圍。
常見錯誤

客戶常見說法背後，真正想談的是什麼

客戶說法背後真正關心你怎麼接
你們比別人貴他在比總成本，或只是想先探底價。先問對方在比哪些條件，不要直接跳到降價。
這是最後價格他在測你還有沒有空間。把話題拉回量、付款、包裝或交期交換。
所有供應商都給 OA他想先把風險轉回供應商。先談交易歷史與信用額度，不要直接跟。
別家交期更快他可能在測試你是否願意加急或拆批出貨。先問優先數量和目標日，再回到成本與可行性。
情境應用

四個高頻談判話術

  • 客戶壓價：「如果要到這個價格，我們需要同步調整條件，例如 MOQ、付款方式或包裝方案，這樣才有機會評估。」
  • 客戶要更短交期：「我們可以先幫您確認加急可行性，但需要先知道優先數量與目標日期，再評估是否會產生加班或空運成本。」
  • 客戶要求 OA：「OA 可以討論，但通常建立在穩定交易記錄或信用保險基礎上。我們可以先從較小額度或縮短帳期開始。」
  • 客戶要索賠：「我們先把事實和責任界面確認清楚，再一起討論最合理的補救方案，避免先處理價格、後面責任卻不一致。」
常見錯誤

三種談判戰術要識別

戰術對方怎麼做你怎麼應對
沉默戰術你說完後，對方沉默不語，等你先開口讓步。不要急著填沉默。說完讓對方消化，沉默 5–10 秒也沒關係。
Good Cop / Bad Cop一個人強硬、一個人示好，輪番施壓讓你覺得「示好的那位」是自己人。識別策略，只和有決策權的人談，不被角色分工帶著走。
蠶食策略快要成交時，對方突然追加小要求：「再加點包裝升級」「再幫我們配合做個認證」。一旦察覺，直接說「需要重新評估整體條件」，讓對方知道加項不是免費的。
💡 這三種戰術共同點：讓你在心理上覺得已經「快成交了」，所以願意再退一步。識別出來，你就不會被帶走。
常見錯誤

哪四件事不能在第一輪就讓步

項目為什麼不能先鬆比較安全的做法
付款條件一鬆就直接影響現金流與放貨控制。先從小額度、短帳期或分階段條件開始談。
交期超出產能承諾後，後面會連帶影響品質與物流。先確認優先數量與可行方案，再談加急條件。
DDP / 門到門責任目的地稅費與清關風險可能遠超想像。先改成 DAP 或 FCA，再看服務能力是否成熟。
模具所有權 / 修改責任一旦講模糊，後面會變成無止盡免費修改。把 ownership、驗收節點與改模責任寫進合同。
情境應用

讓步節奏示例

讓步不是直接降價，而是把讓步和條件綁在一起：

  • 第一次讓步：換更大的 MOQ。
  • 第二次讓步：換較快付款條件或較低放貨風險。
  • 第三次讓步：換長期預測、年度採購承諾或標準化包裝方案。
💡 最健康的談判結果不是「誰壓過誰」，而是每次讓步都換到一個對自己有價值的條件。
情境案例

談判案例：什麼叫做守住底線又不把話說死

案例1：客戶要求年降 8%

這不是單純價格問題，而是整個年度條件重算。

先問年量、付款、材料替代與工藝調整空間，再決定能否部分消化。

案例2：客戶要 DDP 但沒有固定進口流程

看起來像服務要求，實際是把稅費和清關風險一起丟給你。

先改談 DAP 或分段報價，確認目的國服務能力後再決定是否承接。

案例3：客戶說其他供應商可以更快交樣

商務郵件通用結構：5 句就能把事情講清楚

  1. Purpose：第一句直接說這封信的目的，例如確認條件、追蹤回覆、通知異常

貿易合規處理流程

  1. 先確認產品定義：汽車內飾、外飾、發光件、電子模組或單純塑膠件，分類不同會影響稅率。
  2. 請工程或產品確認材質與功能：ABS、PC+ABS、PP、是否有 LED、線束、驅動板、塗裝或電鍍。
  3. 請貨代或報關行初步判斷 HS Code，不要由業務自己憑感覺填。
  4. 確認目的國關稅、301 / 反傾銷 / 原產地要求，並把成本反映到報價條件。
  5. 確認客戶是否要求 REACH、ELV、VOC、禁用物質、供應鏈追溯或 ESG 問卷。
  6. 把最終採用的 HS Code、Incoterm、付款條件、文件清單寫進 PI 或合同，避免出貨前才爭議。
⚠️ 高報/低報發票價格、雙版本發票、虛假原產地都不能配合。這不是談判技巧，是海關與法律風險。

進階情境案例

案例1：美國客戶說關稅太高，要求降價25%

不要直接用降價吸收關稅。先確認HS Code、進口商、Incoterm和關稅由誰負擔，再討論工藝、包裝、MOQ或長約換成本優化。

建議回應：我們可以共同拆解landed cost，但關稅屬目的國進口成本，需和產品單價分開討論。

案例2：歐洲客戶要求ESG問卷三天內完成

先回覆收到並確認截止時間，內部找EHS、品質、採購與人事一起填。不要由業務自己猜數字。

建議回應：我們會協調內部資料，先提供可確認項目，需核准的數據會標註並補交。

案例3：客戶要求低報發票

這是紅線。不能因為想成交就答應，否則會造成出口與進口海關風險，也可能影響公司信用。

建議回應：We are sorry, but we must declare the actual transaction value in compliance with customs regulations.

案例4：EV新創問發光格柵能不能便宜做

先拆功能：發光面積、亮度、動態效果、防水、法規、線束、控制方式。便宜不是先降價，而是先降低功能複雜度。

建議回應：我們可以提供標準版、部分發光版與完整動態光效版三種成本方案。

出貨前合規檢查清單

  1. 確認 HS Code 與申報要素正確，避免錯誤分類造成補稅或罰款
